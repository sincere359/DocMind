# MiMo Orbit 申请表 — 项目描述

## 项目名称
DocMind — AI 文档智能助手

## 项目类型
应用（Web Application）

## 项目定位
DocMind 是一款基于 RAG（检索增强生成）技术的企业级文档智能分析平台，让用户能够与文档"对话"，实现智能问答、自动摘要、关键信息提取和多文档对比分析。核心价值：**将静态文档变为可交互的动态知识库**。

## AI 使用场景

### 1. 智能文档问答（核心场景）
用户上传 PDF/TXT/Markdown/DOCX 文档后，系统自动解析分块、向量化索引，用户即可基于文档内容进行自然语言问答。MiMo V2.5 Pro 负责深度推理生成精准回答，同时标注信息来源片段编号，有效抑制幻觉。支持流式输出，逐 token 实时响应，交互体验流畅。

**技术流程**：文档解析 → 滑动窗口分块（500字/块，50字重叠）→ text2vec-base-chinese 向量化 → FAISS 内积检索 Top-5 相关片段 → 注入 Prompt 作为参考文档 → MiMo V2.5 Pro 推理生成回答

### 2. 自动文档摘要
支持三种摘要风格：简短（3-5句话概括）、详细（含核心论点、数据支撑、结论建议）、学术（研究背景、方法、发现、结论）。长文档采用分层摘要策略：先分块摘要再合并，确保信息完整不丢失。MiMo V2.5 Pro 以 JSON 结构化输出摘要，包含标题、要点、关键词、文档类型等字段。

### 3. 关键信息提取
自动从文档中提取：实体（人名/机构/地点/日期）、关键数据（数值+含义+单位）、术语定义、行动项、核心结论、风险挑战、建议推荐。MiMo V2.5 Pro 以 JSON 格式输出，可直接对接下游系统。长文档采用分段提取+去重合并策略。

### 4. 多文档对比分析
选择两个文档后，AI 自动分析共同点、差异维度，并生成对比结论。适用于合同对比、报告版本差异、竞品分析等场景。MiMo V2.5 Pro 深度推理确保对比全面客观。

## 底层模型依赖

### 当前使用 MiMo 模型
- **mimo-v2.5-pro**：用于所有核心推理任务（问答生成、摘要、信息提取、文档对比），温度 0.1-0.3，确保输出准确可控
- **mimo-v2.5**：用于辅助快速任务（长文档分块子摘要、分段信息提取），温度 0.5，速度优先

### 模型选择理由
1. **中文能力**：DocMind 主要服务中文文档场景，MiMo V2.5 的中文理解和生成能力突出
2. **推理深度**：Pro 模型的思维链推理能力确保文档分析的准确性和逻辑性
3. **成本优势**：Token Plan 模式下 MiMo 性价比远超 GPT-4/Claude
4. **OpenAI 兼容**：SDK 兼容降低集成成本，无需额外适配

## 技术栈详情

| 层级 | 技术 | 用途 |
|------|------|------|
| 前端 | Streamlit | 交互式 Web UI |
| RAG 引擎 | 自研 | 检索增强生成核心 |
| 向量化 | sentence-transformers (text2vec-base-chinese) | 中文文本嵌入，768 维 |
| 向量存储 | FAISS (IndexFlatIP) | 高效余弦相似度检索 |
| LLM | MiMo V2.5 Pro / V2.5 | 推理生成 |
| 文档解析 | pdfplumber + python-docx | PDF/TXT/MD/DOCX |
| SDK | OpenAI Python SDK | 兼容 MiMo API |
